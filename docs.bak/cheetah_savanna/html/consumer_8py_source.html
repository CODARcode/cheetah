19  """Runner that assumes a homogonous set of nodes. Now only support only
20  node based limiting (although process limiting can be emulated by setting
21  process_per_node=1 and max_nodes=max_procs).
22 
23  Threading model: assumes there could be multiple producer threads calling
24  add_pipeline, e.g. if using a dynamic job submission model based on
25  results of previous jobs. Pipelines and each Run in a pipeline are all
26  executed in separate threads, so their notification callbacks execute in
27  separate threads, and their threads must be joined before exiting. The
28  stop and kill_all methods could be called from any of the producer,
